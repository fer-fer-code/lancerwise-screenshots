# ─────────────────────────────────────────────────────────────────────────────
# LemonSqueezy env vars — paste-ready template
# ─────────────────────────────────────────────────────────────────────────────
# Set all 5 in Vercel → Project Settings → Environment Variables.
# Scope: Production + Preview (NOT Development — local .env.local gets these
# manually for test purchases).
#
# Type: all "Encrypted" (Sensitive). None should be "Plain Text".
# Prefix: NO NEXT_PUBLIC_ on any of these — server-side only.
#
# Order matters — set 1 → 5 because checkout/webhook routes throw `Error
# missing` on first request if any are absent.
# ─────────────────────────────────────────────────────────────────────────────

# ─── 1. API key (server-only — never expose client-side) ───────────────────
# Source: LS Dashboard → Settings → API → "Create API key"
# Choose: "All permissions" scope
# Format: starts with `eyJ0eXAi...` (LS uses JWT-style keys)
# Test mode vs Live mode: separate keys exist; Preview = test key, Production = live key
LEMONSQUEEZY_API_KEY=<paste from LS Dashboard → Settings → API>

# ─── 2. Store ID (single numeric ID) ───────────────────────────────────────
# Source: LS Dashboard → Stores → "View" → URL contains numeric ID
#         e.g. https://app.lemonsqueezy.com/stores/12345/dashboard → ID=12345
# Format: integer, ~5-6 digits
# Same value in Production AND Preview (single store covers both)
LEMONSQUEEZY_STORE_ID=<paste store ID number>

# ─── 3. Pro tier variant ID ────────────────────────────────────────────────
# Source: LS Dashboard → Products → "Pro plan" → Variants → copy ID
# Create the product first: Name="Pro", Price=$15/month, Recurring=monthly
# Format: integer
# Same value in Production AND Preview
LEMONSQUEEZY_VARIANT_PRO=<paste Pro variant ID>

# ─── 4. Team tier variant ID ───────────────────────────────────────────────
# Source: LS Dashboard → Products → "Team plan" → Variants → copy ID
# Create the product first: Name="Team", Price=$39/month, Recurring=monthly
# Format: integer
# Same value in Production AND Preview
LEMONSQUEEZY_VARIANT_TEAM=<paste Team variant ID>

# ─── 5. Webhook signing secret ─────────────────────────────────────────────
# Source: LS Dashboard → Webhooks → New webhook → set Endpoint URL =
#         https://www.lancerwise.com/api/lemonsqueezy/webhook
#         Subscribe to events: order_created, order_refunded, subscription_*
#         After creating, click "Show signing secret" and copy
# Format: ~32-char random string (hex or base64)
# Used in `src/app/api/lemonsqueezy/webhook/route.ts` for HMAC SHA-256
# verification of incoming `x-signature` header
LEMONSQUEEZY_WEBHOOK_SECRET=<paste webhook signing secret>
