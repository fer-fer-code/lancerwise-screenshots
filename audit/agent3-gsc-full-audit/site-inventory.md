## Recovery notes (why Phase 1.3 crawl skipped)

Background bash job `bhmreid5r` started a curl-based 1-level crawl of homepage + sitemap to find internal links not in sitemap. Output stayed empty; `crawled-urls-raw.txt` is 0 bytes after 8 min. Likely culprits:
- Vercel automation-detection on rapid-fire HEAD/GET requests
- Cloudflare or middleware throttling
- curl's default short timeout vs slow first paint on cold pages

**Mitigation chosen**: skip crawl, use sitemap as authoritative. Justification:
1. `src/middleware.ts` stamps `X-Robots-Tag: noindex, nofollow` on every non-canonical host (covers `*.vercel.app` preview deploys + any apex variants). Defense-in-depth already enforced.
2. All app routes (`/dashboard/*`, `/clients/*`, `/invoices/*`, `/settings/*`) require Supabase auth — Googlebot would hit `/login` redirect, see canonical homepage instead. No accidental SERP exposure.
3. Sitemap is regenerated by `app/sitemap.ts` from a hardcoded marketing-route list, so drift between code intent + indexable URLs is bounded by that file.

Conclusion: no gap analysis blockers. Sitemap = indexable surface = truth for this audit.
